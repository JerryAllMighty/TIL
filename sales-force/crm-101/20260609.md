[개요]
- 간만에 == 과 eqauls 연산자 비교하면서 옛날에 자바 딥다이브 추억이 새록새록 떠오르길래 다시 한 번 정리해보려한다.
- String 클래스, 원시 타입, 참조 타입 비교하면 금방 이해된다.
- 예시는 강사님이 주신 예시를 활용할 수 있는데, 정리는 나만의 방식으로 다시 다듬어봤다

[내용]
- ==과 equal 은 어떻게 다른지
> == 은 스택의 값을 비교하는 것이라고 생각하면 편함
> 원시 타입은 값(밸류)가 들어있으니 값을 비교
> 참조 타입은 주소 값이 들어 있으니 주소를 비교하는 것이고, 따라서 값이 같아도 주소 값이 다르면 다른 결과로 판단
- 그럼 참조값은 값 비교 어떻게 하나?
> Object 클랫의 equals는 따로 재정의하지않으면 == 비교. 따라서 사용자 정의 객체라면 같이 작성해줄 필요 있음.
> String, Integer, List 등은 이미 정의되어 있기에 equals를 활용하면 값 비교 가능
```
String str1 = new String("abc");
String str2 = new String("abc");
```
> 이 경우 str1 == str2의 케이스는 서로 다른 주소값을 가지기에 false이다
> 하지만 값은 같기 때문에 equals는 true이다
```
String str1 = "abc"; // (1)
String str2 = "abc"; // (2)
```
> 이 케이스는 얼핏보면 서로 다른 주소값일 것이라고 생각되지만, 스트링 클래스는 따옴표로 할당시 JVM 힙 영역 내부의 특별한 메모리 공간인 스트링 컨스탄트 풀을 찾아감.
> 따라서 이미 있는 값이라면 str2처럼 값이 할당될 떄 이미 같은 값으로 존재하는 str1의 메모리 값을 할당.
> 결과적으로 동일한 힙 메모리 주소를 가진다.
