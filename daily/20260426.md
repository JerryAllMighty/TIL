[요약]
- mysql의 서버는 mysql 엔진과 스토리지 엔진이 있음
- 이 스토리지 엔진은 쉽게 말하면 손과 발임
- mysql엔진이 머리라서, 얘는 여러개의 스토리지 엔진을 가짐
- 그래서 innoDb 외의 인증이나 전문 검색 파서 같은 다른 플러그인도 활용 가능함
- 이 머리와 손은 핸들러를 통해 소통함


- 스레드는 포그라운드, 백그라운드가 있음
- 포그라운드는 클라이언트 수만큼 존재함. 종류에 따라 버퍼나 캐시에 데이터를 가져오려다 없으면 직접 디스크에 데이터를 쓰기도 함.

- 메모리 할당 및 사용 구조는 크게 글로벌과 세션으로 나뉨
- 글로벌은 모든 스레드에 의해 공유됨
- 로컬은 세션 메모리라고도 하는데, 클라이언트 스레드가 사용하는 메모리 공간이라고 생각하면 된다.
