- Features and Object Access
각 프로필별 사용자 capa를 어떻게 결정하는지,
Permission Set과 Permission Set Group의 목적을 아는 것,
어떻게 profile, permission set, permission set group이 필드 접근을 제어하는지 설명하는 것을 목표로 하는 단
1. Profiles
> Organization > Object > Record Field 순으로 접근 제어
> 좀 특이한 건 이떄 CRED라고 표현을 하더라. Edit이더라, Update가 아니라. 그래서 전통 개발과는 조금 다르게 표현하는 부분이 있구나 싶었다.
> User가 생성되면 Profile이 하나씩 매칭되는게 이게 Permission 등의 권한 관련 정보를 담고 있음
> standard profile 종류가 많은데,
> 이 중 Minimun access salesforce 는 최소한의 권한, 유저가 필수 기능만 가지고 시작할 수 있음을 의미
> Profile은 커스텀 가능 : clone 혹은 기존 것 수정하는 방법 가

2. Permission Sets
- 사람마다 필요한 권한이 모두 다를 때, Permission Set은 특정 유저에게 추가적인 권한을 줄 수 있당
- Permission Set을 하나로 묶은 Permission set group을 활용할 수 있다
- 만약 특정 인원만 레코드를 삭제 못하게 막는다거나의 기능이 필요하면 Muting Permission을 줄 수 있다
- 어드민은 기본적으로 디폴트라 생각하면 되는 Manage On Profile와 추가 개념인 Manage On Permission Set을 생각하면 된다.


3. Field-Level Security
핃드별로 접근 권한을 제어할 필요가 있을 떄, 프로필이나 permission set으로 관리 가
User Access Policy를 통해 복잡한 사용자간의 복잡한 권한 관계를 쉽계 풀 수 있음
