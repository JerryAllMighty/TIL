[문제]
- 기존의 group by는 결과 집합의 모양이 바뀐다. 
- 나도 사용했을 때는 집계 함수등을 group by랑 썼을 때 count등은 전체 집합의 결과를 하나의 row로 리턴해주었었다.
[해결]
- 윈도 함수는 결과 집합을 그대로 유지 가능하고, 하나의 레코드 연산에 다른 레코드의 칼럼 값 참조 가능하다.

윈도우 함수 자체는 무조건 써도 될까?
[문제]
- mysql 8.0부터 도입이되었는데, 아직 인덱스를 이용한 최적화가 부족함
- 예를 들면,  
> select max(from_date) Over(partition by emp_no) as max_from_date from salaries;
> select max(from_date) from salaries group by emp_no;
- 이렇게 윈도우와 집계 함수를 각 사용하는 쿼리가 있다고 하면 실행계획에서는 큰 차이가 나는데,
윈도우 함수는 테이블의 모든 레코드 건수만큼의 결과를 만들어야하고,
group by는 유니크한 emp_no 별로 레코드 1건씩만 만들면 된다.그래서 레코드 건수에서 차이가 난다.
- 또 윈도우 함수는 정렬을 사용하며, 인덱스를 풀스캐한다.
- 실행시간도 윈도우 함수는 3.2초, 그룹바이는 1.2초이다. 
[해결]
- 이 부분은 개인 해결 가능한 부분은 아니라서, 사용 기준을 정립하는 것이 좋다
- 배치는 윈도우 함수 사용 무방하다. 온라인 트랜잭션 처리에서는 사용 지양해야한다. 
- 소량의 레코드는 사용 무관하다.
