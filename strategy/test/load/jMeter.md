[고려 포인트]

- warm-up 과정 필요한지
- 측정하고자 하는 상황이 어떤 식으로 부하를 받는 상황인지
  부하, 스트레스, 장기 안정성, 스파이크
- 각 상황에 맞는 프로필
동시 사용자수, 초당 요청수, ramp-up/ramp-down 시간, 테스트 총 duration, think time (사용자 대기 시간)
- 데이터 중복/경합 (동시성)
주문, 결제처럼 같은 리소스를 여러 유저가 동시에 건드릴 때의 케이스를 어떻게 만들지
- 성능 지표
  합격과 실패의 판단 기준
  응답 시간, 에러율, 처리량.
- 인프라
어느 환경에서 테스트할지 로컬/개발/스테이징
- 모니터링
테스트 중 어떤 지표를 어떤 툴을 사용해서 볼 것인지 (CloudWatch, CpuUtilization)
