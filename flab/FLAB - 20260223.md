[서두]
- 오늘도 이력서 작성에 박차를 가했다
- 솔직하게 하루종일 보냈다 생각보다 이렇게 시간 잡아먹을 줄이야
- 그래도 그냥 적지않고, 읽는 사람을 위하면서도 나를 어필하기 좋은 구조를 생각하려 애를 썼다
- 자소서를 작성했고, 집중했던 포인트들을 간략하게 요약해보겠다

[자기소개서]
- 먼저 글의 키워드를 정의하려했다. 처음에는 소제목을 다른 자소서를 조금 참고했다. 
어떤 걸 어필하는지 좀 보려고 했다. 그래서 소제목을 많이 좀 간추렸다.
- 협력, 자동화, 테스트 코드 등등을 간추려서 뽑아냈고, 일단 장황하게 써보았다.
하지만 강조하고자 하는 포인트, 각 소제목에 맞게 어필을 하고 내용을 각인시키려면 조금 더 글을 짧고, 
포인트를 앞에 두는게 좋을 것 같더라. 그래서 구조를 다시 재정의했다.
1. 소제목 (강조하고자하는 포인트)
2. 실제 내가 겪은 사례
3. 결과
- 혹시 기존에 잘 알려진 기법이 있을까 싶어서 찾아보았다.
- S - Situation (상황)
  T - Task (과제/목표)
  A - Action (내가 한 행동)
  R - Result (결과)
- 자소서에 상황을 모두 쓰면 너무 장황해해지니 간단하게 상황을 적고, 각 소제목이 Task가 되겠다.
- 이걸 나만의 방식으로 흡수한다고 가정하면, 다음부터는 좋은 방식의 자소서는 이렇게 정리할 수 있을 것 같다.

**
각 소제목 (목표): 내가 드러내고 싶은 나의 역량이 될 것 같다.

실제 내가 겪은 사례 (상황): 간단하게 설명을 도울 수 있을 것 같다. 

내가 한 행동: 이건 내가 겪은 행동을 적을 수도 있는데, 보통 경력 기술서에 이런 건 많이 적을 것 같다. 
이 상황에서 내가 배운 것을 나중에 어떻게 습득하려 노력했는지 녹이는 것도 좋을 것 같다. 

결과: 짤막하게, 되도록이면 정량적인 수치로 표현하면 더 좋을 것 같다고 생각했다.
