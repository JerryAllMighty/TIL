[왜 해야하나 페이징? 언제 하나?]
- 리소스의 한계 때문에. 자바 힙 메모리는 생각보다 금방 찬다.
- DB 커넥션 점유 및 네트워크 부하. DB 커넥션을 데이터 가져오는 동안 계속 붙잡고 있는데, 이게 커지면 
전체 시스템의 응답 속도에 영향을 미침.
- 트랜잭션 시간이 길어져서 DB나 WAS 설정에 의해 강제 종료될 수 있음.

[조회하면 왜 자바 힙 메모리가 차나?]
- DB 정보가 자바의 객체가 되면 관련 메타데이터, 연관관계 객체까지 모두 메모리에 올라온다. 
따라서 실제 크기보다 몇 배는 커지게 되고, 또한 GC는 대량 데이터 조회시 현재 사용중인 데이터는
청소를 못한다. 그래서 OOM이 난다.
- JPA 사용시 영속성 컨텍스트에 1차 캐시에 조회한 데이터를 보관한다. 10만건만 되어도 꽉 찰 확률이 높다. 

[page vs slice]
- page는 전체 건수를 세는 count 쿼리가 한 번 더 날라간다. 성능상 불리할 수 있다.
