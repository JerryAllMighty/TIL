<details>
<summary> 외부 API와의 연동 시에 왜 원시 타입보다는 래퍼 클래스를 사용하는 것이 더 안전할까요? </summary>
<div markdown="1">
보통 외부 기관 등에 의해서 동작하기에 개발자가 제어할 수 없는 영역이라 값을 보장할 수 없기에 래퍼 클래스가 더 유리합니다.
또한 원시 타입은 기본형을 리턴하기에 명확한 원인 파악이 어려울 수 있습니다. 실제 값이 0인지, 전송이 원할하지 않아 실패한 값인지 판단이 어렵습니다. 하지만 null 체크를 통하면 명확하게 의도된 예외처리를 할 수 있는 부분도 판단의 근거가 됩니다.
</div>
</details>
