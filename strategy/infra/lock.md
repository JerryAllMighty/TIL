[애플리케이션 레벨 락]
- synchronized / reentrantLock : 특정 코드 블록이나 메서드에 단 하나의 스레드만 접근하도록 보장
- Atomic 클래스 : cas 방식. AtomicInteger 등은, 하드웨어 수준의 cas 연산을 통해 락 없이도 안전하게 값을 갱신
> 단순 조회수 증가나 공유 카운터 변수 관리 시 사용

[DB 레벨 락]
- 낙관적 락 : 데이터 충돌이 적을 것으로 예상. @Version 체크를 통해 수정 시점에 데이터 정합성 체크.
- 비관적 락 : 데이터 충돌이 빈번, selecr for update를 통한 물리적인 락을 건다.

[선택 기준]
- 성능 면에서는 낙관적 락이 우세, 충돌이 낮아 롤백 비용이 크다면 비관적 락 선택
